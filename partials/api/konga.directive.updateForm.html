updateForm
directive in module konga

Description

Handles rendering and operations of updation/creation forms. It connects to the EntityUpdateController through several operations for field mapping, change notifictions, etcetera.

The first operation is to receive fields, categories and fieldSets, that are used to render the update layout. These elements are provided to the view determined on the field's formType.

Views are based on rawInput directives, who controls validation and mapping, along with all other field operations.

Usage

as element:
<update-form
       metadata="{Object}"
       entity="{Object}"
       creating="{Boolean}"
       [fields="{Array}"]
       on-update="{function()}"
       [on-change="{function()}"]>
</update-form>

Parameters

ParamTypeDetails
metadataObject

Entity The metadata of the entity being managed

entityObject

The entity object to map values to

creatingBoolean

Whether the mode is creation.

fields
(optional)
Array

Optionally, include a set of fields instead of retrieving all from the entity (default when this value is not set).

onUpdatefunction()

Define the behavior for field-update procedures - when an input's value changes. This function signature shall be similar to the default updateEntityField method, present on the EntityUpdateController.

onChange
(optional)
function()

Optional change tracker to know whether the field has changed
