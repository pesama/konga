Konga's API Reference

Welcome to Konga's API Reference. Here you got all the documentation, about everything Konga does and gives you. Here you can see a brief introduction about what you can find in each section.

Metadata

This section defines everything konga-metadata contains, and what each property and parameter means, and does:

  • Definitions: Definitions contain individual or grouped properties to configure your metadata. Some of these properties are interpreted by the ui giving a more meaningful context to the forms and the interactions engaged with them. Here you got a set of all konga-metadata definitions:

    • Application: Defines your app globally, and the root level configuration.
    • Entity: Contains the metadata for each entity your app needs to manage.
    • Field: Includes the definition of each field, with all the required parameters.
    • Action: Defines each customisation entry-point within your metadata.
    • Trigger: Configure automated actions to be executed every time a field changes.
    • ConfigurationParam: Define the configuration parameters for your metadata items.
    • ShowConfiguration: Configure the rendering configuration for a field within a form scope.
    • FieldType: Define the rendering appeareance for a field within a form scope.
    • DataType: Configure the typology of a field - i.e. data type, allowed values, filtering, querying...
    • FieldSet: Setup groups among your fields within entities, mostly for category/group-based form types.
    • Validation: Define the required validation options for your fields.
    • Validator: Define custom validators to be appended to your validation configuration.

  • Enumerations: Enumerations contain the set of allowed parameters for the properties bound to them. Here you have all existing enumerations.

    • AccessModes: Define the possible visibility options for an entity within your app.
    • DataTypes: Contains the existing data-types Konga can manage natively.
    • FieldTypes: Define the possible rendering appeareances fields can have.
    • FormScopes: Define the existing scopes Konga has for building the different kinds of forms.
    • FormStyles: Configuration options for the field layout within a form.
    • FormTypes: Contains all possible form rendering configurations for your forms.
    • Multiplicities: Define the possible multiplicities - quantity of items per input - for a field.
    • TriggerMatches: Define the possible trigger configurations for the value to verify matches.
    • TriggerMoments: Contains all possible moments where a field verify its triggers.
    • ValidatorTypes: Includes the possible validation modes for the data within your fields.
  • Generators: Documents the available methods for generating your metadata.
  • Permissions: Learn here how to engage your Konga app with a rich permission system to restrict access at both entity and field levels.

Standards

Documentation on standard features. Standard features are enough for building a full-working app, connected to an API. These are the main standard features:

  • Modeling: Define how your metadata is interpreted to build your Apps and Forms. It includes examples on every available form scope, and links to related inner elements.
  • Users: Define how your user management integrates into Konga, and the features you can leverage using this system.
  • Tools: Documents the set of tools Konga uses to manage your metadata, and the macros you can use to save time while developing.
  • Operations: Documents all built-in operations for managing your application and its interactions.

Controllers

  • KongaController: This is the main controller of Konga, and gives your application access to all Konga's features and operations.
  • EntitySearchController: Engaged on search and results form scopes, it controls all required tasks and behaviors to allow full functioning of search panes and result tables.
  • EntityUpdateController: Used on updates, it's responsible for building the update form, controlling all operations engaged within the form and its fields, and all the tasks involved with them.
