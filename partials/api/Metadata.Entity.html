Entity
service in module Metadata

Description

All elements within your data model are represented using this object. Entities define data structures for applications with options (rendering, operability, fields...)

Usage

new Entity(name, superClass, label, shortLabel, access, searchable, createable, editable, deleteable, searchType, resultsType, updateType, updateType, detailsType, searchStyle, resultsStyle, updateStyle, detailsStyle, apiPath, categories, fieldSets, configuration, fields, actions, overrideDefaults);

Parameters

ParamTypeDetails
namestring

Name for your entity. This unique name will identify your entity across all your application. All relationships to an entity will be declared using its name. Be wise while defining your entity names.

superClassstring

Entity hierarchy. By defining a superClass to one entity (being the value of the superClass the name of the inheritance) Konga will create entities based on the child's model, along with all the inheritance tree entities.

labelstring

Label for your entity. The String you configure here (or the placeholder for translated content) will identify the entity within your app.

shortLabelstring

Short version of the label. Just if you need a shorter name to render it somewhere small.

accessObject

AccessMode

Accessibility of the entity. You can set the entity to be public or hidden (system entities).

searchablestring

Defines the search capabilities for the entity. If the value is null, search will be deactivated. For restricting it to some permission, put its name here.

createablestring

Defines the creation capabilities for the entity. If the value is null, creation will be deactivated. For restricting it to some permission, put its name here.

editablestring

Defines the edition capabilities for the entity. If the value is null, edition will be deactivated. For restricting it to some permission, put its name here.

deleteablestring

Defines the deletion capabilities for the entity. If the value is null, deletion will be deactivated. For restricting it to some permission, put its name here.

searchTypeObject

FormType

Defines the mode on which the search form will be rendered. See the FormTypes documentation for more details about the possible modes.

resultsTypeObject

FormType

Defines the mode on which the results table will be rendered. See the FormTypes documentation for more details about the possible modes.

updateTypeObject

FormType

Defines the mode on which the update form will be rendered. See the FormTypes documentation for more details about the possible modes.

updateTypeObject

FormType

Defines the mode on which the update form will be rendered. See the FormTypes documentation for more details about the possible modes.

detailsTypeObject

FormType

Defines the mode on which the details pane will be rendered. See the FormTypes documentation for more details about the possible modes.

searchStyleObject

FormStyle

Defines the styling of the search pane. (How to structure the layout label/field, and among fields). See the FormStyles documentation for more details.

resultsStyleObject

FormStyle

Defines the styling of the results table. See the FormStyles documentation for more details.

updateStyleObject

FormStyle

Defines the styling of the update form. (How to structure the layout label/field, and among fields). See the FormStyles documentation for more details.

detailsStyleObject

FormStyle

Defines the styling of the details pane. (How to structure the layout label/field, and among fields). See the FormStyles documentation for more details.

apiPathstring

Defines the path of this entity within the API's endpoint.

e.g. if your api's path is /api and your entity's /api/path then the apiPath for your entity will be 'path'.

categoriesArray

Categorisation for the entity. This is not used by Konga directly, but let's you categorise your entities to configure things (such as menus) quickly.

fieldSetsObject

FieldSet

FieldSets bound fields with similara categorisation together, to be rendered in the same view. This is used on several form modes.

configurationArray

ConfigurationParam

Maps all configuration for the entity. Here could go from native configuration parameters to any key/value item you need for doing your custom stuff.

fieldsArray

Field

Fields within the entity.

actionsArray

Action

Custom behaviors to be bounded to your entity.

overrideDefaultsArray

Action

Custom actions to override native behaviors.
