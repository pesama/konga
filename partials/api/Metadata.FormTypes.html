FormTypes
service in module Metadata

Description

Form types define how to render the form fields layout.

Properties

  • CASCADE

    Render the fields in a cascade - i.e. one after another.

  • CATEGORIZED_CASCADE

    Render the form as a field-cascade, but grouping the fields by 'category', and labeling each category. Depending on the form scope, some configuration parameters are needed:

    • SEARCH_USE_CATEGORY: SEARCH Define the categories to use on search (one or more).
    • RESULTS_USE_CATEGORY: RESULTS Define the categories to use on results (one or more).
    • HIDE_CATEGORY_HEADER: RESULTS If this parameter is associated with a rendered category, its header will be hidden.

  • CUSTOM

    When standard form types don't fit your needs, you can declare the form to have a CUSTOM view, attaching any view you want to the form's behavior. For using this you need to provide a ConfigurationParam depending on the form scope:

    • SEARCH_CUSTOM_VIEW: SEARCH Custom view for search (value must be a valid mapped view).
    • UPDATE_CUSTOM_VIEW: UPDATE Custom view for updates (value must be a valid mapped view).

  • CUSTOM_TABBED

    The same as the TABBED type, but each tab is rendered using a custom view. You need to define the fieldSet view param to a valid mapped view for this to work.
