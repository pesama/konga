Field
service in module Metadata

Description

Fields represent each property of the entities. Field configuration includes data typing, rendering and validation options, custom behaviors, automated triggers...

Usage

new Field(name, label, shortLabel, type, isId, isKey, isLabel, multiplicity, categories, fieldSet, defaults, priority, actions, overrideDefaults, searchConf, validation, searchable, quickSearch, showInResults, showInUpdate, showInDetails, editable, required, configuration, triggers, linked);

Parameters

ParamTypeDetails
namestring

Name of the field.

labelstring

Label for the field.

shortLabelstring

Short version of the label.

typeObject

DataType

Field's data type. This value stores not only the type name but also other information, such as valid options (useful for combo boxes), and other options used for reference fields.

isIdBoolean

The entity's field with this flag set to true will represent the unique ID of such entity. This field's value is used to query APIs and other tasks requiring computed-identification of the entity. This field normally represents the key of the element in any persistence system, and it's hidden on every form.

(default: false)

isKeyBoolean

If this property is true on a certain field, the entity will be humanly-identifiable using such field. While isId represents computer-identification (primary keys), isKey is another unique identification, but for human quick-targeting (such as people's IDs, or car's plate numbers). If your entity does not have a value like this, you can annotate the same field with both isId and isKey.

(default: false)

isLabelBoolean

Entity's field with this flag to true represents the name of the entity. In a example Human entity, the field marked with isLabel would be fullName.

(default: false)

multiplicitystring

Multiplicities

Defines the multiplicity of the element.

categoriesArray

Categorisation of the field. This is used on certain form modes, and you can leverage this info anywhere on your custom code.

fieldSetstring

Defines the group this field belongs to. It's set up as a String, but it must be the name of a fieldSet defined at entity level. This is used in TABBED, and CUSTOM_TABBED forms, and you can easily pick all the fields of a fieldSet using a filter (everywhere in your custom code).

(default: null)

defaultsstring

Default value of the field. It's used on scaffoldings of entities and queries. It's always set up as a string, and is cast based on the field's type.

(default: null)

priorityObject

Priority

Defines the [sorting] priority for the fields within the forms. You can set up same priority for all form scopes, or define separate priority for each one.

(default: null)

actionsArray

Action

Array of custom actions associated with the field. Depending on the field type chosen for the field, the actions will be rendered (and engaged and executed) differently. See the field types documentation for more details.

overrideDefaultsArray

Action

Array of custom actions to override field's defaults. Depending on the field type chosen for the field, there would be a different set of actions available to override. See the field types documentation for more details.

searchConfObject

SearchConf

Defines the configuration parameters for using this field as a filter in queries. You can define multiplicities, type of search, etcetera. See the SearchConf documentation.

validationObject

Validation

Defines the validation options for the field. Depending on the field type chosen for the field, some validation options could not be applied.

searchableObject

ShowConfiguration

Defines the search capabilities for the field. This property defines search permissions, inner fields (for reference items), and some other configuration parameters.

quickSearchObject

ShowConfiguration

Defines the search capabilities for the field on the quickSearch pane. This property defines search permissions, inner fields (for reference items), and some other configuration parameters.

showInResultsObject

ShowConfiguration

Defines the results pane configuration for the field. This property defines viewing permissions, inner fields (for reference items), and some other configuration parameters.

showInUpdateObject

ShowConfiguration

Defines the update form configuration for the field. This property defines viewing permissions, inner fields (for reference items), and some other configuration parameters.

showInDetailsObject

ShowConfiguration

Defines the details form configuration for the field. This property defines search permissions, inner fields (for reference items), and some other configuration parameters.

editableObject

ShowConfiguration

Defines the edition permissions for the field. The other fields of the property are not used.

requiredBoolean

Defines whether the field is required. A required field won't let the form submit untill it's not filled up.

configurationArray

ConfigurationParam

Configuration parameters for the field

triggersArray

Trigger

Triggers for the field. Triggers are automated tasks susceptible of executing when a field changes. See the trigger docs for more info.

linkedObject

Linked

Defines a dependence for this field to others. With this you can engage a custom action to the link, to be executed once the linked fields change (e.g. generating a fullName attribute based on a name and a surname).
