resultTable
directive in module konga

Description

The resulTable is in change of result rendering, into an interactive data table. It generates a column layout using the configuration set up in the fields' metadata.

Fields and categories' metadata are fetched, to determine the column layout to generate. If you configured any field to be rendered as complex, another process is needed, who splits up your root field into all the inner fields you need for your layout.

Form Type

You can customize the appeareance of your table, by setting up the property formType of the field's metadata, for results form scope. There are built-in views to render category headers on top of the fields.

Sorting

If field's metadata configures the field as sortable, a dropdown/caret will show on table header to allow user to launch sorting. The resultTable calls the submitSorting method of the EntitySearchController with the sorting field and mode, and the controller handles it from there.

Usage

as element:
<result-table
       metadata="{Object}"
       entities="{Array}"
       update-entity="{function()}"
       submit-sorting="{function()}">
</result-table>

Directive info
