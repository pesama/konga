ValidatorTypes
service in module Metadata

Description

Defines the different validation methods for evaluating the field's value against data-integrity methods.

Properties

  • DATE_GE

    Date validation. Validates that your date is Greater or Equal than the one provided.

  • DATE_GT

    Date validation. Validates that your date is Greater Than the one provided.

  • DATE_LE

    Date validation. Validates that your date is Lower or Equal than the one provided.

  • DATE_LT

    Date validation. Validates that your date is Lower Than the one provided.

  • EXACT_MATCH

    Validate the value using an exact match. Value must be identical to the matching value.

  • FUNCTION

    TODO Custom action to validate your input.

  • INLINE

    Inline validation. If you use this method the text you include must be a syntax-valid expression that returns true or false. You can append the value into your inline expression using the String '{value}'

  • RANGE

    Validate the input with a range. It simply verifies your value is between the minimum and maximum defined. Validations on this case are written as 'M-X' - where M represents the minimum value allowed, and X the maximum.

  • REGEXP

    Validate the value using regular expressions. These expressions must be declared as Strings, as they are validated using ng-pattern (and it converts automatically the values provided to RegExp).
