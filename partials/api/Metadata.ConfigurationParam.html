ConfigurationParam
service in module Metadata

Description

A configuration param stores a raw property, which is normally attached to a Konga structure element (i.e. applications, entities and fields). Configuration is interpreted into Konga via a configuration manager, who reads all hierarchy configurations and applies the required value.

Native configuration parameters

You have all native configuration parameters here, as the properties of the element.

Custom configuration parameters

Konga's configuration manager intention is not only to fulfill native Konga configurations, but also for giving you an expansion layer to put aditional configuration into elements, that could enrich your app in any way you want.

An useful usage for this feature would be to define schemas for your entities and fields, that you could easily retrieve and render in your views, giving you full semantic compliance of your objects, and a better understanding of your app to machines. Well, it's just an example.

Usage

new ConfigurationParam(key, value);

Parameters

ParamTypeDetails
keystring

Key for the configuration parameter

valuestring

Value for the configuration parameter
