What is Konga?

Konga is an application engine, whose purpose is to improve development experience when dealing with form-based apps, by reducing required times and efforts drammatically - up to 80%. It does so by introducing a new concept into our classical development tasks: Metadata modeling.

Metadata modeling

Metadata modeling helps you define your elements, and convert them into an object, interpreted by the UI to generate your app. So instead of creating the forms manually for each entity, you would just define the metadata, and let Konga do the rest.

The metadata object definitions cover all contextual information about the data of your app - e.g. identification, data typing or validation. Definitions are applied to these elements:

  • Application: Is the root of your app, and you can provide it with a name, and certain configuration.
  • Entities: Define the elements of your app - e.g. Car, Person, ... - and can configure identification, allowed operations - CRUD - and appearance.
  • Fields: Describe the properties of any entity, and can control identification, data typing, appearance, validation and operations.

The metadata is stored in a JSON object, which is the input for the UI to start working with Konga. To simplify metadata definitions, we've provided you with some generators that let you write metadata in an easier way, and get it transformed to JSON.

Konga UI

Konga UI is an AngularJS library, that includes a set of standards that will help you create a fully-working UI to manage your data with very little coding. It contains all required features to read the metadata, and generate search & results and update forms automatically.
